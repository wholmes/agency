BrandMeetsCode — GTM web container import (bmc-web-container-import.json)
============================================================================

WHAT'S IN THE FILE
------------------
- User variables: Cookie - bmc_vid; Data Layer vars for page_location, page_title,
  page_path, currency, value, form_id (used when you add GA4 Event tags yourself).
- Triggers: All Pages; Custom Event virtual_page_view; Custom Event generate_lead.
- Tags: One Google tag (googtag) with measurement ID G-CZWQMF2S67, server_container_url
  https://brandmeetscode.com/metrics, send_page_view true, client_id = {{Cookie - bmc_vid}}.

IMPORTANT
---------
You likely ALREADY have a Google tag and variables in your workspace. Importing can CREATE
DUPLICATES. Prefer importing into an EMPTY workspace or choose Merge and rename conflicts.

HOW TO IMPORT
-------------
1. tagmanager.google.com → Web container → Admin → Import container.
2. Choose workspace → Merge (recommended) or Overwrite (destructive).
3. Select bmc-web-container-import.json.
4. Resolve conflicts: if names clash with your existing Google tag, rename the imported
   tag or skip duplicates.

BEFORE PUBLISHING
-----------------
- Replace measurement ID in the JSON before import if yours is not G-CZWQMF2S67; or edit
  the tag after import.
- Edit server_container_url if your first-party metrics URL differs.
- Optional: Add two GA4 Event tags (type GA4 Event in UI) firing on triggers
  CE - virtual_page_view and CE - generate_lead, linked to the same Google tag / GA4,
  using the DL-* variables for event parameters. This repo pushes those dataLayer keys.

IF IMPORT FAILS
---------------
GTM's JSON format varies slightly by UI version. Create one tag in the UI, then use
Admin → Export container and compare structure to this file. Official reference:
https://support.google.com/tagmanager/answer/6106997

Known enum: container usageContext must be WEB (uppercase), not web — required by import.

TROUBLESHOOTING (PRODUCTION CONSOLE / GTM PREVIEW)
-------------------------------------------------
You may see messages like:
  Invalid Ecommerce event name "gtm.init_consent" / "gtm.js" / "gtm.dom" / "gtm.load"
  [WARN] data.event "form_start" is not a valid event
  [ERROR] ttq not found
  TikTok Pixel - Invalid Event Name Format

What they mean:

1) gtm.* events — These are normal Google Tag Manager lifecycle events (container boot).
   They are NOT ecommerce events. Warnings usually come from Tag Assistant / Preview debug
   mode or from a tag that validates event names against GA4 ecommerce. They are noisy but
   harmless unless you misconfigured a tag to treat every event as a purchase.

2) form_start / form_submit — From GA4 Enhanced Measurement (automatic form tracking). Same
   story: not "ecommerce" events; a strict validator or wrong tag wiring causes the warning.
   Fix in GTM: do not send all gtag events to TikTok/Meta as ecommerce; use explicit triggers.

3) ttq not found + TikTok "Invalid Event Name" — The TikTok browser pixel (ttq) is missing
   OR a GTM tag is forwarding GA4/gtag events to TikTok without filtering. TikTok only
   accepts specific event names (e.g. CompleteRegistration, SubmitForm, ViewContent).
   Forwarding "gtm.init", "gtm.dom", "form_start", etc. causes rejections.
   Fix in GTM: TikTok tag(s) should fire only on dedicated triggers (e.g. Custom Event
   generate_lead), not on "All pages" + every GA4 event, and ensure the TikTok base snippet
   loads before dependent tags (or use server-side Events API only).

4) connect.facebook.net ... ERR_BLOCKED_BY_CLIENT — Almost always an ad blocker or strict
   privacy extension; not a site bug.

5) /api/contact 500 — Application / Resend delivery, unrelated to GTM. Check Vercel logs
   for Resend errors and verify From domain + RESEND_API_KEY.
